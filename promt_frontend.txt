Tolong buatkan komponen React (komponen Modular/Modal) menggunakan Tailwind CSS yang khusus menangani fitur **Menambah dan Menghapus Custom Action (Hak Akses)** pada suatu Menu.

**Skenario / Flow UI yang Diinginkan:**
1. Anggaplah pengguna sedang berada di halaman pengaturan Role dan sedang melihat deretan Menu. Saat pengguna menyorot atau mengeklik salah satu menu spesifik (misalnya menu "Laporan"), terdapat sebuah tombol **"Tambah Action Baru"**.
2. Jika tombol "Tambah Action Baru" diklik, sebuah **Modal / Pop-up Form** muncul meminta inputan:
   - **Kode Action** (Input teks, contoh: `export_pdf`, `approve_data`)
   - **Deskripsi Action** (Input teks, contoh: `Export Data User ke PDF`)
3. Di dalam Modal terdapat tombol "Simpan" untuk memanggil API Penambahan Action, dan tombol "Batal" untuk menutup Modal.
4. Selain penambahan, di UI daftar action yang sudah ada pada menu tersebut, sediakan **Tombol Hapus (Icon Trash / Silang merah)** di sebelah label tiap-tiap action. Jika di-klik, panggil API Penghapusan Action.

**Spesifikasi API Integrasi Backend:**

**1. API Menambah Action Custom pada Menu**
- Endpoint: `POST /api/sys-group-permissions/add-action`
- Header: `Authorization: Bearer <token>`
- Body Request:
```json
{
  "id_sys_group": 1, 
  "id_sys_menu": 15, 
  "code": "export_pdf",
  "description": "Export Data User ke PDF"
}
```
*(Asumsi: `id_sys_group` adalah ID grup yang sedang diedit, dan `id_sys_menu` adalah ID dari menu di mana tombol tambah action ditekan).*

**2. API Menghapus Action dari Menu**
- Endpoint: `POST /api/sys-group-permissions/remove-action`
- Header: `Authorization: Bearer <token>`
- Body Request:
```json
{
  "id_sys_menu_permission": 32
}
```
*(Asumsi: `id_sys_menu_permission` merupakan ID unik masing-masing relasi action yang melekat pada Menu tersebut dan disematkan di tombol Hapus).*

Mohon buatkan kodenya. Siapkan struktur state React untuk Modal (open/close), form input, maupun indikator loading saat tombol "Simpan" atau "Hapus" sedang mengeksekusi request ke API.
